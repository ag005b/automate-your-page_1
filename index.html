Unit 1: The Web and Structured Documents

Lesson 1: Web Basics and HTML

How the Web Works

The Internet is a global system of interconnected computer networks that interchange data. When an individual goes to a website like www.espn.go.com, their computer sends a HTTP Request to a server. The server finds the appropriate HTML document and sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen. Watch this video for a better explanation.

HTML

HTML is an ancornym that stands for Hypertest Markup Language. HTML documents make up the majority of the content on the web. HTML documents contain text content which describes "what you see" and markup which describes "how it looks". This video provides a good overview.

Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags. This video differentiates the two.

Inline vs. Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Lesson 2: Using HTML to Create Structured Documents

The "tree-like structure" of HTML

The DOM (Document Object Model) is a way to refer to HTML elements as objects. The "tree-like structure" of the page comes about because elements can be nested within each other. In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...

Text Editors (for programming)

Programmers use special text editors (i.e. Sublime Text). Some text editors will automatically generate a closing HTML tag when you write an opening tag evidently making the programmer's job a little bit easier.

Lesson 3: Addition of CSS to HTML Document

CSS

CSS stands for Cascading Style Sheets and it is code written to control the "style" of the HTML elements. You can include CSS styling by:

1. Write CSS in the "head" of your HTML.
2. Link your HTML to a separate CSS file. (This method is more organized when working on larger projects).
Benefits of CSS Avoiding Repetition

Using CSS to avoid repetition is very important for a number of reasons. The programmer becomes more efficient because they do not have to write the same thing multiple times. Hence there is less code to write and it's more readable. If the programmer is copying and pasting certain style attributes into many HTML elements it can possible lead to errors when the programmer decides to modify something.

With CSS, the programmer can assign multiple elements the same class and then specify a style for that class. in only one location. Giving every one of those element the same Consistent look and being able to easily modify the style and avoiding repetition.

Unit 2: Telling Computer What to Do

Lesson 1: Intro To Programming

Computers

Computers can be programmed to carry out a set of instructions.

A computer program, is an exact sequence of instructions, written to perform a specified task with a computer. Some examples of computer programs are web browsers and simple print statements.

Programming Language

Programming is the core of Computer Science. A programming language is a formal constructed language designed to communicate instructions to a computer. Examples of programming language are Python, C++, Java and SQL.

Compiler

Compiler - a program that producers another program. It transforms source code written in a programming language (the source language) into another computer language (the target language).

Python

Python is a programming language. When you write Python code and press "Run", a Python Interpreter converts the code you wrote as a set of instructions that the computer itself can understand and execute.

Debugging

Debugging is a methodical process of finding and reducing the number of bugs, or defects, in a computer program, to eveidently have it behave as expected.

Here are some simply steps to follow while debugging your program:

Examine error messages when programs crash
The last line of Python Tracebacks will tell you what went wrong. Reading backwards from there will tell you more about where the problem occurred.
Work from example code
If your modified code doesn't work, comment it out and do step-by-step modifications to the example code until it does what you want.
Make sure examples work
Just because you find example code doesn't mean it will work in your system. Check the example code you're using to make sure it behaves the way you expect.
Check (print) intermediate results
When your code doesn't crash, but doesn't behave as expected, add print statements to your program to see where in the code things stop behaving correctly.
Keep and compare old versions
When you have a working version of your code, save it before you add to the code. This will give you something to go back to if you introduce too many new bugs.

Lesson 2: Variables and Strings

Define Variable in Python

A variable gives names value that may change. It gives a name to a value. If test_variable is a variable with a value of 3, then the following code would print out 9:

print test_variable * test_variable

Meaning of Assigning a Varible and How to do so

A programmer can assign a value to a variable by writing the following code:

test_variable = 10

The variable test_variable will have the value of 10. The programmer can change the value of the variable by re-assigning it to a different value at any time.

Difference between 5+2=7 and test_variable=7

The equal sign in the line, 5+2=7 means "is the same as".

The equal sign in the line, test_variable=7 means "takes the value of".

Ways Variables are Useful

There are many ways variables prove to be useful to programmers:

  • The value of important data can be stored using variables.
  • They improve the code readability by using names that make sense to humans.
  • They give us a way to change the value of something (like in the line counter = counter + 1
Difference between 4 + 4 and "4" + "4"

In Python, 4 is a number while "4" is a string.

The code 4 + 4 would give 8.

The code "4" + "4" would give "44".

Strings

A string is a sequence of characters surrounding by quotes. A string can be surrouded by single quotes (') or double quotes (").The only requirement is if it starts with a sigle or double quote it needs to end with a single or double quote respectively.

Python does not allow you to concatenate a string and an interger. However, strings can be multiplied. The following code:

print '!' * 3

Will display: !!!

Indexing String

Python uses square brackets to extracts parts of the string. Characters of the strings are indexed starting with the number zero.

For example:

test_variable = 'Udacity'

print test_variable[0] will display U

print test_variable[3:6] will display city

Lesson 3: Input -> Function -> Outout

What is a Function?

Functions usually "take in" data, process it, and "return" a result. Once a function is written, it can be used over and over and over again. Functions can be "called" from the inside of other functions.

For example, a function named cubed might take a number as input and produce the cube of that number as output.

The Difference in Making and Using a Function

Functions are made by starting a line of code with the keyword def and then giving a function name followed by the function parameters in parentheses. These parameters will eventually be replaced by actual values when the function is used (called).

In the "body" of the function, we write the code that specifies what to do with the input parameters. For example the following code could be the definition of a function called cubed:

def cubed(x):
answer = x * x * x
return answer

To use a function, we write the name of the function followed by the value(s) we want to give it in parentheses. Like this:

print cubed(3)
>>>27
How do Functions Help Programmers Avoid Repetiton?

Functions are tools that programmers can create and reuse time and time again. Once you've defined a function once, you never have to define it again.

What happens if a Fucntion Does Not Have a Return Statement?

The return keyword tells Python exactly what the function should produce as output. If a function doesn't have a return statement, then you'll get behavior like this:

def minus_four(x):
answer = x - 4

new_number = minus_four(7)
print new_number
>>>None

Lesson 4: Decisions and Repititon

If and While

Control flow statements (like 'if' and 'else') let us execute different blocks of code depending on some condition. While loops let us repeat a certain block of code many times until a certain condition is met.

if, then, else: These three Python keywords let us tell Python what code we want it to execute in different situations.

while loops: A while loop will repeat the indented block of code beneath it as long as the condition on the first line (right after 'while') is true.
